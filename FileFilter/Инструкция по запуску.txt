# Инструкция по запуску:

## Требования:
    • Java версия 17 или выше
    • Maven версия 3.6.0 или выше (для сборки)

## Сторонние библиотеки:
        <dependency>

            <version>1.18.30</version>
        </dependency>
        <dependency>
            <groupId>org.junit.jupiter</groupId>

            <scope>test</scope>
        </dependency>

## Сборка проекта
    Клонировать проект
    Собрать проект с помощью Maven (mvn clean package)
    После сборки в директории target появится файл file-filter.jar

## Запуск утилиты:
    Запустить терминал (в среде разработки, Git Bash или другой)
    Перейти в корневую директорию проекта (cd FileFilter)

### Параметры программы задаются при запуске через аргументы командной строки:
-o задает путь для результатов (по умолчанию берется текущая папка),
-p задает префикс имен выходных файлов (по умолчанию без префекса),
-a задает режим добавления (по умолчанию идет перезапись файлов).
Выбор статистики производится опциями -s (короткая, по умолчанию) и -f (длинная).
Остальные параметры – имена входных файлов, не менее одного.

### Простой запуск (входные файлы находятся в папке data, которая находится в корне проекта)
java -jar target\file-filter.jar data\in1.txt data\in2.txt

### Запуск без параметров (получаем справку по использованию параметров)
java -jar target\file-filter.jar

### С указанием выходной директории и префикса
java -jar target\file-filter.jar -o data\res -p result_ data\in1.txt data\in2.txt

### Тоже самое с полной статистикой и режимом добавления в существующие файлы
java -jar target\file-filter.jar -o data\res -p result_ data\in1.txt data\in2.txt -f -a

## Особенности реализации
### Определение папок
     По умолчанию файлы (входящие/исходящие) располагаются в текущей папке (из которой запускается утилита).
     Если в параметрах перед директорией стоит слеш, то она считается от корня диска.
     Если без слеша или точка + слеш, считается путь относительно от текущей папки.
     Для работы с другим диском нужно прописывать "абсалютный" путь (с указанием диска).
     Наклон слеша не влияет на работу.
### Определение типа данных:
     Строка считается целым числом, если содержит от 1 до 256 цифр без пробелов со знаком или без.
     Строка считается вещественным числом со знаком или без, если содержит цифры,
     разделенные точкой или экспоненциальное выражение с (e/E) в конце.
     Ограничение количества цифр от 0 до 128 перед точкой и после точки.
     Хотя бы одна цифра должна быть до или после точки.
     От 1 до 3 цифр у экспоненты  со знаком или без.
     В остальных случаях строка считается строкой.
### Обработка ошибок:
     Программа продолжает работу при ошибках чтения отдельных строк
     При ошибках чтения файла, программа продолжает обработку других файлов
     При ошибках в параметрах, программа работает с параметрами по умолчанию
### Производительность:
     Используется буферизованный ввод/вывод
     Файлы для записи открываются один раз и держатся открытыми до завершения программы
### Кодировка
     Для чтения и записи используется кодировка UTF-8
