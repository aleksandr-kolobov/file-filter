# Утилита фильтрации содержимого файлов

При запуске утилиты в командной строке подается несколько файлов,
содержащих в перемешку целые числа, строки и вещественные числа. В качестве
разделителя используется перевод строки. Строки из файлов читаются по
очереди в соответствии с их перечислением в командной строке.

Задача утилиты записать разные типы данных в разные файлы. Целые числа в
один выходной файл, вещественные в другой, строки в третий. По умолчанию
файлы с результатами располагаются в текущей папке с именами integers.txt,
floats.txt, strings.txt. Дополнительно с помощью опции -o нужно уметь задавать
путь для результатов. Опция -p задает префикс имен выходных файлов.

Например -o /some/path -p result_ задают вывод в файлы
/some/path/result_integers.txt, /some/path/result_strings.txt и тд.

По умолчанию файлы результатов перезаписываются. С помощью опции -a
можно задать режим добавления в существующие файлы.

Файлы с результатами должны создаваться по мере необходимости. Если какого-
то типа данных во входящих файлах нет, то и создавать исходящий файл,
который будет заведомо пустым, не нужно.

В процессе фильтрации данных необходимо собирать статистику по каждому типу
данных. Статистика должна поддерживаться двух видов: краткая и полная. Выбор
статистики производится опциями -s и -f соответственно. Краткая статистика
содержит только количество элементов записанных в исходящие файлы. Полная
статистика для чисел дополнительно содержит минимальное и максимальное
значения, сумма и среднее. Полная статистика для строк, помимо их количества,
содержит также размер самой короткой строки и самой длинной.

Статистику по каждому типу отфильтрованных данных утилита должна вывести в
консоль

# Инструкция по запуску:

## Требования:
    • Java версия 17 или выше
    • Maven версия 3.6.0 или выше (для сборки)

## Сторонние библиотеки:
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <version>1.18.30</version>
        </dependency>

## Сборка проекта
    Клонировать проект
    Собрать проект с помощью Maven (mvn clean package)
    После сборки в директории target появится файл file-filter.jar

## Запуск утилиты:
    Запустить терминал (в среде разработки, Git Bash или другой)
    Перейти в корневую директорию проекта (cd FileFilter)

Параметры программы задаются при запуске через аргументы командной строки:
-o задает путь для результатов
-p задает префикс имен выходных файлов
-a задает режим добавления 
Выбор статистики производится опциями -s (короткая) и -f (длинная)
Остальные параметры – имена входных файлов, не менее одного.

### Простой запуск (входные файлы находятся в папке data, которая находится в корне проекта)
java -jar target\file-filter.jar data\in1.txt data\in2.txt

### Запуск без параметров, выводится справка по использованию
java -jar target\file-filter.jar

### С указанием выходной директории и префикса
java -jar target\file-filter.jar -o data\res -p result_ data\in1.txt data\in2.txt

### Тоже самое с полной статистикой и добавлением в существующие файлы
java -jar target\file-filter.jar -o data\res -p result_ data\in1.txt data\in2.txt -f -a

По умолчанию файлы с результатами располагаются в текущей папке (из которой запускается утилита).
Если перед директорией стоит слеш, то она считается от корня диска.
По умолчанию файлы с результатами перезаписывается.

## Особенности реализации
### Определение типа данных:
        Строка считается целым числом, если содержит цифры без пробелов
        Строка считается вещественным числом, если содержит точку или экспоненту (e/E)
        В остальных случаях строка считается строкой
### Обработка ошибок:
        Программа продолжает работу при ошибках чтения отдельных строк
        При ошибках чтения файла, программа продолжает обработку других файлов
        При ошибках в параметрах, программа работает с параметрами по умолчанию
### Производительность:
        Используется буферизованный ввод/вывод
        Файлы для записи открываются один раз и держатся открытыми до завершения программы
